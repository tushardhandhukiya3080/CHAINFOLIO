Smart Contracts & dApps

This is where blockchains go from "digital money" to "programmable money." Smart contracts let you put agreements into code that runs exactly as written — with no middleman.

What is a smart contract?

A smart contract is a program stored on the blockchain that runs automatically when conditions are met. Think of a vending machine: insert the right amount, and it guarantees you get the snack — no cashier needed. Smart contracts do that for digital agreements.

Automatic — executes itself when triggered; no human approval step.
Immutable — once deployed, the code generally can't be changed (so bugs are costly).
Transparent — anyone can read the code and verify what it does.
Trustless — you trust the code, not the counterparty.

Ethereum & Solidity

Ethereum was the first blockchain built to run smart contracts. It acts like a giant shared "world computer" called the EVM (Ethereum Virtual Machine). The most popular language for writing these contracts is Solidity.

Here's a tiny Solidity contract — read the comments to follow along:

// A simple contract that stores a number anyone can read or update
pragma solidity ^0.8.0;

contract SimpleStorage {
    uint256 public storedNumber;   // state saved on-chain

    // "set" changes the value (costs gas — it writes to the chain)
    function set(uint256 _number) public {
        storedNumber = _number;
    }

    // "get" just reads the value (free — no state change)
    function get() public view returns (uint256) {
        return storedNumber;
    }
}
Why it matters: Once deployed, this code lives forever on Ethereum. Anyone, anywhere can call set and get — no server, no company, no downtime.

From contracts to dApps

A dApp (decentralized application) is a normal-looking app whose backend logic runs on smart contracts instead of a company's servers. The pieces:

🖥️

Frontend

A regular website (HTML/CSS/JS or React) — exactly the skills you're practicing now.

🔌

Wallet connection

Libraries like ethers.js or web3.js let the site talk to the user's wallet (e.g. MetaMask).

📜

Smart contracts

The on-chain "backend" that stores data and enforces the rules.

This is your bridge to real Web3 dev: the front end of a dApp is just a website like the ones on this site. Add a wallet connection and a few contract calls, and you've built a dApp.

Real-world uses

  • DeFi — lending, trading, and earning interest without banks.
  • NFTs — provable ownership of digital art, tickets, and items.
  • DAOs — communities that vote on and run shared treasuries via code.
  • Supply chain & identity — tamper-proof records of where things came from.
